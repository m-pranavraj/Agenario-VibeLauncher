3 Chaos Actions the Swarm Executes (That LLMs Can't Mimic)
1. The "Impatient User" Double-Click Test
The Scenario: A user clicks a "Submit Payment" or "Generate AI Image" button three times rapidly because the server is taking 2 seconds to respond.

The Reality: In poorly structured apps, this triggers three separate database writes, charges the card multiple times, or crashes the API route.

Agenario's Execution: The agent actively double-clicks every primary button while tracking database states or network responses, flagging duplicate request failures.

2. State Disconnection & Hydration Checks
The Scenario: A user opens a dashboard link directly without going through the onboarding flow, or clicks "back" midway through a complex form multi-step process.

The Reality: Global state (like Zustand or Redux) clears out, resulting in the dreaded white screen of death or Cannot read properties of undefined (reading 'user').

Agenario's Execution: The browser agent intentionally jumps deep into private routes with empty local storage or sessions to see exactly where the app breaks cleanly versus where it completely crashes visually.

3. Dynamic Form Abuse (Beyond Simple Input)
The Scenario: A user uploads a broken 0-byte file, an emoji-laden string into a phone number field, or spaces instead of a name.

The Reality: Frontend validation might catch basic errors, but if the unhandled data hits the server components or database, it throws a 500 Internal Server Error.

Agenario's Execution: Agents aggressively fuzz inputs inside the actual browser rendering tree, observing console logs for unhandled frontend exceptions.
